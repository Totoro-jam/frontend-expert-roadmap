Layout 抖动 vs 仅 Composite —— 性能对比

开启 DevTools Performance 录制后再点按钮,你会看到完全不同的火焰图。

核心规则

  1. 读取 layout 信息 (`offsetWidth` `getBoundingClientRect` `getComputedStyle`) 之前如果有 pending 的样式改动,浏览器**必须先 layout**
  2. 所以反复 read → write 会触发 1000 次 layout,而不是 1 次
  3. 解决:批量先 read 全部,再 write 全部(FastDOM 库的核心思想)
  4. 更优:用 `transform` / `opacity` 让 compositor 接管,完全跳过 layout/paint
