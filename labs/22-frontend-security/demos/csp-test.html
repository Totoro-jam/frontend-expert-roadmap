CSP 测试页

这页 CSP:

script-src 'self' 'nonce-DEMO_NONCE' 'strict-dynamic'
style-src 'self' 'unsafe-inline'
img-src 'self' https: data:
connect-src 'self' https://httpbin.org
object-src 'none'
frame-ancestors 'none'

预期:带 nonce 的 inline script ✅ 跑;不带 nonce 的 ❌ 被拒;跨域 fetch 看 connect-src。

1. Inline script(无 nonce) → 应该被拦截

2. Inline script(有 nonce) → 应该执行

3. eval → 默认应该被拦截(没 unsafe-eval)

4. Image — img-src 'self' https: data:

5. fetch 跨域 — connect-src 限制

6. iframe — frame-ancestors 'none' 不让别人嵌我

这页设了 frame-ancestors 'none',你试着在另一个 HTML 里 <iframe src="这个文件"> → 浏览器会拒绝渲染。

7. 内联 style

如果这行是蓝色 → style-src 'unsafe-inline' 生效

8. javascript: URL — 应该被拦

点击试 javascript: URL

9. 动态 import script(带 nonce 链)

10. 看 console

F12 打开 Console,CSP 违规会输出:

Refused to execute inline script because it violates the following Content Security Policy directive: "script-src 'self' 'nonce-...' 'strict-dynamic'". Either the 'unsafe-inline' keyword, a hash ('sha256-...'), or a nonce ('nonce-...') is required to enable inline execution.

如何在生产用

  1. 用 HTTP 头(不是 meta)
  2. 每个请求生成新 nonce(中间件)
  3. 先用 Content-Security-Policy-Report-Only 灰度
  4. 用 report-uri / report-to 收集违规
  5. strict-dynamic 替代逐 origin 白名单
