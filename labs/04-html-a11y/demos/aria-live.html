aria-live —— 让屏幕阅读器知道 UI 更新了

开启 VoiceOver(Mac:Cmd+F5),点按钮听一下,体会三种 live region 的差异。

常用 live region 类型

  • aria-live="polite" — 用户停止操作时朗读。**默认选这个**。
  • aria-live="assertive" — 立即打断当前朗读。**只用于严重错误**。
  • role="status" — = polite,但更语义化(状态信息)
  • role="alert" — = assertive(警告)
  • role="log" — 日志区,屏幕阅读器知道是追加

真实场景

  • 表单提交后的 success/error toast
  • 搜索框下方「找到 N 条结果」
  • 异步加载完成「加载完成」
  • 购物车「已添加到购物车」
  • WebSocket 断开重连提示

反例

// ❌ 视觉 toast 只显示文字,屏幕阅读器完全感知不到
<div class="toast">保存成功</div>

// ✅ 配合 live region
<div class="toast">保存成功</div>
<div class="sr-only" aria-live="polite">保存成功</div>
