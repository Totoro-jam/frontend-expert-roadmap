Resource Hints 速查

优先级 + 适用场景

提示做什么使用时机优先级
dns-prefetch DNS 解析 本页/下页要请求其他域 最低
preconnect DNS + TCP + TLS ~4-6 个最重要的第三方源 中
preload 立刻下载本页资源 LCP 图、关键 CSS、首屏字体 最高(Highest)
modulepreload ESM 模块预加载+解析+连同依赖 ESM 入口 高
prefetch 空闲时下,给下一页 下一页资源(链接已知) Lowest
Speculation Rules prerender 整页预渲染 下一步几乎确定 (>80%) —

常见陷阱

  • preload 字体不加 crossorigin → 浏览器下两次
  • preload 不写 as= → 浏览器忽略
  • preload 太多 → LCP 资源排队,反而变慢
  • preconnect > 6 个 → TCP 连接抢占,反向劣化
  • prerender 误用 → 浪费数 MB 流量、隐私问题
  • 字体一律 preload + font-display:swap
  • LCP 图一律 fetchpriority="high"

验证

DevTools → Network → 右键表头勾上 Priority,看到:

  • Highest(preload critical)
  • High(stylesheet, font, image with fetchpriority=high)
  • Medium(image, script defer)
  • Low / Lowest(prefetch, image lazy)
