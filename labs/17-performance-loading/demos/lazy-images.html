图片加载策略对比

打开 DevTools → Network → 看 priority 列和 timing,体会区别。

1. Eager(默认):立刻全部加载

所有 <img> 一进 DOM 就开始下,小图无所谓,LCP 大图必须这个

<img src="..." />

2. LCP 图:加 fetchpriority="high"

告诉浏览器:这是首屏最重要的图,优先调度

<img fetchpriority="high" />

3. Native lazy loading

浏览器原生支持,无需 JS,viewport 临近时才下载

↓ 往下滚 ↓
<img loading="lazy" decoding="async" />

4. IntersectionObserver(自定义控制)

需要更精细控制时(占位、动效、加载追踪)

↓ 再滚 ↓
const io = new IntersectionObserver((entries) => {
  entries.forEach(e => {
    if (e.isIntersecting) {
      const img = e.target
      img.src = img.dataset.src
      io.unobserve(img)
    }
  })
}, { rootMargin: '200px' })  // 提前 200px 加载

5. LQIP / BlurHash:占位 + 模糊过渡

先显示 base64 缩略图,大图加载完再切换

↓ 最后滚 ↓

真实方案:next/image / @plaiceholder/next / blurhash / thumbhash

对比表

策略适合LCP 友好实现成本
